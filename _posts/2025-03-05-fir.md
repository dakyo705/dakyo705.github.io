## iOS 개발 관련 기본 질문
1. **Xcode란 무엇인가요?**
   - Xcode는 Apple의 통합 개발 환경(IDE)으로, iOS 및 macOS 애플리케이션을 개발하는 데 사용됩니다. 코드 작성, 디버깅, UI 디자인 등을 지원합니다.

2. **Swift의 특징은 무엇인가요?**
   - Swift는 안전성, 성능, 그리고 현대적인 문법을 갖춘 프로그래밍 언어입니다. 타입 안전성과 메모리 관리를 통해 오류를 줄이고, 간결한 문법으로 개발 효율성을 높입니다.

3. **iOS 앱의 생명주기란 무엇인가요?**
   - iOS 앱의 생명주기는 앱이 시작되고 종료되는 과정을 포함합니다. 주요 상태는 Not Running, Inactive, Active, Background, Suspended입니다.

4. **MVC 패턴이란 무엇인가요?**
   - MVC(Model-View-Controller)는 애플리케이션을 세 가지 구성 요소로 나누어 관리하는 디자인 패턴입니다. Model은 데이터, View는 사용자 인터페이스, Controller는 둘 사이의 상호작용을 담당합니다.

5. **Delegate 패턴이란 무엇인가요?**
   - Delegate 패턴은 객체 간의 통신을 위한 디자인 패턴으로, 한 객체가 다른 객체에 특정 작업을 위임하는 방식입니다. iOS에서 자주 사용됩니다.

## 기술적 질문
6. **Swift에서 옵셔널(Optional)이란 무엇인가요?**
   - 옵셔널은 값이 있을 수도 있고 없을 수도 있는 변수를 나타내는 타입입니다. 이를 통해 nil 값을 안전하게 처리할 수 있습니다.

7. **Closure란 무엇인가요?**
   - Closure는 코드 블록을 캡슐화하여 변수처럼 사용할 수 있는 기능입니다. 함수와 유사하지만, 변수에 할당하거나 인자로 전달할 수 있습니다.

8. **Auto Layout이란 무엇인가요?**
   - Auto Layout은 다양한 화면 크기와 방향에 맞춰 UI 요소의 위치와 크기를 자동으로 조정하는 시스템입니다. 제약 조건을 사용하여 레이아웃을 정의합니다.

9. **Swift에서 Error Handling은 어떻게 하나요?**
   - Swift에서는 `try`, `catch`, `throw` 키워드를 사용하여 오류를 처리합니다. 함수에서 오류를 발생시키면 호출하는 쪽에서 이를 처리해야 합니다.

10. **Core Data란 무엇인가요?**
    - Core Data는 iOS에서 데이터 모델을 관리하고 영구 저장소에 데이터를 저장하는 프레임워크입니다. 객체 그래프 관리와 데이터 영속성을 제공합니다.

기술 관련 질문
Swift 및 iOS 기초
Swift의 주요 특징은 무엇인가요?
Swift는 안전성, 속도, 현대적인 문법을 갖춘 Apple의 프로그래밍 언어입니다. 옵셔널, 타입 추론, 프로토콜 지향 프로그래밍을 지원합니다1.

Value type과 Reference type의 차이점은?
Value type은 데이터를 복사하여 전달하고, Reference type은 메모리 주소를 참조합니다. struct는 Value type, class는 Reference type입니다1.

Optional이란 무엇이며 왜 사용하나요?
Optional은 값이 있을 수도, 없을 수도 있음을 나타내는 타입입니다. 안전한 코드 작성과 null 참조 오류 방지를 위해 사용합니다1.

Guard 문의 용도는 무엇인가요?
Guard는 조건을 먼저 검사하여 조건이 맞지 않으면 빠르게 함수를 종료할 수 있게 해주는 제어문입니다1.

Closure란 무엇인가요?
Closure는 이름 없는 함수로, 간결한 문법으로 코드 블록을 캡처하고 전달할 수 있게 해줍니다1.

iOS 프레임워크 및 아키텍처
UIKit과 SwiftUI의 차이점은 무엇인가요?
UIKit은 명령형 UI 프레임워크이고, SwiftUI는 선언적 UI 프레임워크입니다. SwiftUI는 더 간결한 코드와 실시간 프리뷰를 제공합니다3.

MVC, MVVM 아키텍처에 대해 설명해주세요.
MVC는 Model, View, Controller로, MVVM은 Model, View, ViewModel로 구성됩니다. MVVM은 View와 Model의 분리를 더 강화하여 테스트와 유지보수를 용이하게 합니다1.

Core Data란 무엇인가요?
Core Data는 iOS 앱의 모델 계층 관리를 위한 프레임워크로, 데이터의 저장, 추적, 필터링, 동기화를 지원합니다3.

Auto Layout이란 무엇이며 왜 중요한가요?
Auto Layout은 제약 조건을 기반으로 UI 요소의 크기와 위치를 동적으로 계산하는 시스템입니다. 다양한 화면 크기에 대응하기 위해 중요합니다1.

GCD(Grand Central Dispatch)란 무엇인가요?
GCD는 멀티코어 시스템에서 동시성 프로그래밍을 쉽게 구현할 수 있게 해주는 저수준 API입니다1.

메모리 관리 및 성능
ARC(Automatic Reference Counting)란 무엇인가요?
ARC는 Swift의 메모리 관리 시스템으로, 참조 카운팅을 자동으로 처리하여 메모리 누수를 방지합니다1.

weak와 unowned 참조의 차이점은?
weak는 옵셔널 타입으로 nil이 될 수 있고, unowned는 항상 값이 있다고 가정합니다. 순환 참조 방지에 사용됩니다1.

메모리 누수를 방지하는 방법은?
순환 참조를 피하고, weak나 unowned 참조를 적절히 사용하며, 객체 해제 시 모든 리소스를 정리합니다1.

앱의 성능을 향상시키는 방법은?
코드 최적화, 캐싱 사용, 비동기 처리, 이미지 최적화, 네트워크 요청 최소화 등이 있습니다1.

Instruments 툴을 사용해 본 경험이 있나요?
Instruments를 사용하여 메모리 누수, CPU 사용량, 네트워크 활동 등을 프로파일링하고 최적화한 경험이 있습니다1.

네트워킹 및 데이터 처리
URLSession이란 무엇인가요?
URLSession은 HTTP/HTTPS를 통한 데이터 전송을 위한 API입니다. 네트워크 요청을 생성하고 응답을 처리합니다1.

JSON 파싱을 어떻게 하나요?
JSONDecoder를 사용하여 JSON 데이터를 Swift 객체로 디코딩합니다. Codable 프로토콜을 준수하는 모델을 정의하여 사용합니다1.

비동기 프로그래밍이란 무엇이며 왜 중요한가요?
비동기 프로그래밍은 작업을 백그라운드에서 실행하여 UI의 반응성을 유지하고 사용자 경험을 향상시킵니다1.

Combine 프레임워크에 대해 설명해주세요.
Combine은 시간에 따라 값을 처리하기 위한 선언적 Swift API로, 비동기 이벤트를 위한 통합 방식을 제공합니다3.

RESTful API란 무엇인가요?
RESTful API는 HTTP 프로토콜을 기반으로 리소스에 대한 CRUD 작업을 수행하는 웹 서비스 아키텍처 스타일입니다1.

CI/CD에 대해 알고 있나요?

Unit Testing의 중요성은 무엇인가요?

질문 12: ARC(Automatic Reference Counting)란 무엇인가요?

답변: ARC는 메모리 관리를 자동으로 처리하는 시스템입니다. 객체의 참조 카운트를 추적하여 더 이상 필요 없는 객체를 자동으로 해제합니다. 이를 통해 메모리 누수를 방지하고 개발자가 직접 메모리를 관리하는 부담을 줄여줍니다.

질문 13: Delegate 패턴이란 무엇인가요?

답변: Delegate 패턴은 객체 간의 1:1 통신을 위한 디자인 패턴입니다. 한 객체가 특정 작업을 다른 객체에게 위임하여 처리하도록 합니다. 예를 들어, UITableView에서 delegate를 설정해 셀 선택 이벤트를 처리할 수 있습니다.

질문 14: Closure란 무엇인가요?

답변: Closure는 특정 컨텍스트에서 변수나 상수를 캡처하고 사용할 수 있는 독립적인 코드 블록입니다. Swift에서는 주로 비동기 작업이나 콜백 함수로 사용됩니다. 예를 들어, API 호출 후 데이터를 처리할 때 사용됩니다.

질문 15: Optional이란 무엇이며 왜 사용하나요?

답변: Optional은 값이 있을 수도 있고 없을 수도 있는(nil) 상태를 표현하는 타입입니다. Swift에서 변수가 nil일 가능성을 명시적으로 처리하도록 하여 런타임 오류를 방지합니다. 예를 들어, var name: String?는 name이 String이거나 nil일 수 있음을 나타냅니다.

질문 16: Unwrapping Optional의 방법은 무엇이 있나요?

답변: Optional을 Unwrapping하는 방법은 여러 가지가 있습니다:

Force Unwrapping (!): 값이 확실히 존재할 때 사용하지만, nil이면 크래시가 발생합니다.
Optional Binding (if let / guard let): 안전하게 값을 추출하는 방법입니다.
Nil-Coalescing Operator (??): 기본값을 제공합니다.
Optional Chaining (?.): 중첩된 Optional에서 안전하게 접근합니다.

질문 31: App Lifecycle에 대해 설명해 주세요.

질문 32: Memory Leak이란 무엇인가요?

답변: Memory Leak은 더 이상 필요 없는 메모리가 해제되지 않고 계속 점유되는 상황입니다. 예를 들어, 강한 참조 사이클(Strong Reference Cycle)이 발생하면 ARC가 객체를 해제하지 못해 메모리 누수가 발생합니다.

질문 33: Strong, Weak, Unowned 참조의 차이는?

답변:

Strong: 기본 참조 타입으로, 참조 카운트를 증가시킵니다.
Weak: 참조 카운트를 증가시키지 않으며, 참조하는 객체가 해제되면 자동으로 nil이 됩니다.
Unowned: Weak와 비슷하지만 nil이 되지 않으며, 참조하는 객체가 해제되면 런타임 오류가 발생합니다.

답변: View Controller의 생명주기는 다음과 같습니다:

viewDidLoad: 뷰가 메모리에 로드된 후 호출.
viewWillAppear: 뷰가 화면에 나타나기 직전에 호출.
viewDidAppear: 뷰가 화면에 나타난 후 호출.
viewWillDisappear: 뷰가 사라지기 직전에 호출.
viewDidDisappear: 뷰가 사라진 후 호출.

func binarySearch(_ array: [Int], _ target: Int) -> Int? {
    var left = 0
    var right = array.count - 1
    
    while left <= right {
        let mid = (left + right) / 2
        if array[mid] == target {
            return mid
        } else if array[mid] < target {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    return nil
}

    var charIndex: [Character: Int] = [:]
    var maxLength = 0
    var start = 0
    
    for (i, char) in str.enumerated() {
        if let lastIndex = charIndex[char], lastIndex >= start {

질문 79: 새로운 팀원에게 프로젝트를 설명한다면?

답변: 프로젝트의 목표와 구조를 간단히 설명하고, 주요 기능과 기술 스택을 소개합니다. 코드베이스를 함께 리뷰하며 구체적인 질문을 기다립니다.

질문 80: 동료가 어려움을 겪고 있다면 어떻게 돕나요?

답변: 먼저 그들의 상황을 이해하고, 제가 아는 범위에서 도움을 제안합니다. 함께 디버깅하거나 자료를 공유하며 문제를 해결하려 합니다.

5. iOS 생태계 및 최신 트렌드 관련 질문 (10개)
질문 81: WWDC에서 발표된 최신 기능 중 관심 있는 것은?

답변: 최근 WWDC에서 발표된 SwiftUI 개선 사항과 WidgetKit 확장에 관심이 많습니다. 사용자 경험을 더 풍부하게 만들 수 있는 기능들이라 흥미롭습니다.

질문 82: SwiftUI를 사용해 본 적이 있나요?

답변: 네, 간단한 프로젝트에서 SwiftUI를 사용해 봤습니다. 선언형 방식이 UI를 빠르게 구성하는 데 유용했고, 상태 관리도 직관적이었습니다.

질문 83: iOS 17에서 추가된 기능은 무엇인가요?

답변: iOS 17에서는 스탠바이 모드, 대화형 위젯, NameDrop 같은 기능이 추가되었습니다. 개발자 입장에서는 SwiftData와 같은 새로운 프레임워크도 주목할 만합니다.

질문 84: App Store 심사 과정에서 주의할 점은?

답변: 애플의 가이드라인을 준수하고, 개인정보 보호 정책을 명확히 명시해야 합니다. 또한 앱이 크래시 없이 안정적으로 동작해야 하며, 불필요한 권한 요청을 피해야 합니다.

질문 85: ARKit에 대해 아는 대로 설명해 주세요.

답변: ARKit은 증강현실 앱을 개발하기 위한 애플의 프레임워크입니다. 장면 인식, 모션 추적, 라이트 추정 등을 지원하며, RealityKit과 함께 사용할 수 있습니다.

질문 86: iOS에서 머신러닝을 어떻게 활용하나요?

답변: Core ML을 사용해 머신러닝 모델을 통합할 수 있습니다. 예를 들어, 이미지 분류 모델을 앱에 적용해 사진을 분석하는 기능을 추가할 수 있습니다.

질문 87: WidgetKit을 사용해 본 적 있나요?

답변: 네, WidgetKit을 사용해 간단한 날씨 위젯을 만들어 본 적 있습니다. TimelineProvider를 통해 데이터를 업데이트하고, SwiftUI로 UI를 구성했습니다.

질문 88: iOS에서 보안은 어떻게 고려하나요?

답변: 데이터를 암호화하고, Keychain을 사용해 민감한 정보를 저장합니다. HTTPS를 사용해 네트워크 통신을 보호하고, 앱 내에서 사용자 데이터를 최소화합니다.

질문 89: TestFlight란 무엇인가요?

답변: TestFlight는 앱 배포 전 베타 테스트를 위해 사용하는 애플의 플랫폼입니다. 테스터를 초대해 앱을 배포하고 피드백을 받을 수 있습니다.

질문 90: iOS 개발 트렌드 중 주목하는 것은?

답변: 최근에는 SwiftUI와 Combine을 활용한 반응형 프로그래밍이 주목받고 있습니다. 또한 개인정보 보호와 접근성 강화도 중요한 트렌드라고 생각합니다.

6. 기타 질문 (10개)
질문 91: 좋아하는 iOS 앱은 무엇인가요?

답변: 저는 Notion 앱을 좋아합니다. 깔끔한 UI와 생산성 도구로서의 활용성이 뛰어나서 자주 사용합니다.

질문 92: 버전 관리 도구는 무엇을 사용하나요?

답변: Git을 주로 사용하며, GitHub를 통해 협업합니다. 브랜치 관리와 풀 리퀘스트를 통해 코드 변경 사항을 관리합니다.

질문 93: Xcode의 유용한 기능은 무엇인가요?

답변: Xcode의 디버깅 도구와 Interface Builder가 특히 유용합니다. 또한 Instruments를 통해 성능 분석을 할 수 있는 점이 좋습니다.

질문 94: 코딩 스타일 가이드는 어떻게 따르나요?

답변: 팀에서 정한 가이드라인이 있다면 이를 따르고, 없으면 Swift의 공식 스타일 가이드나 Airbnb 스타일 가이드를 참고합니다.

질문 95: 개인 프로젝트에서 가장 어려웠던 점은?

답변: 네트워크 요청과 데이터 동기화가 어려웠습니다. 비동기 처리와 에러 핸들링을 배우며 해결했고, 이후로 관련 기술에 더 관심을 가지게 되었습니다.

질문 96: 커뮤니티 활동이나 오픈소스 기여 경험이 있나요?

답변: 아직 오픈소스 기여는 없지만, Stack Overflow와 같은 커뮤니티에서 질문을 올리거나 답변을 보며 학습하고 있습니다.

질문 97: 개발자로서 어떤 도구를 자주 사용하나요?

답변: Xcode, Git, Postman, Notion 등을 자주 사용합니다. Postman은 API 테스트에 유용하고, Notion은 학습 노트를 정리하는 데 활용합니다.

질문 98: 최신 iOS 버전에서 주의할 점은?

답변: 새로운 API와 정책 변화를 확인해야 합니다. 예를 들어, 개인정보 보호 정책이 강화되었기 때문에 권한 요청 시 주의가 필요합니다.

질문 99: 앱 성능을 어떻게 측정하나요?

답변: Instruments를 사용해 CPU, 메모리, 네트워크 사용량을 측정합니다. 또한 앱의 시작 시간과 UI 응답성을 체크합니다.

질문 100: 면접관에게 질문이 있나요?

답변: 팀에서 주로 사용하는 기술 스택이나 코드 리뷰 프로세스가 궁금합니다. 또한 신입 개발자를 위한 멘토링 프로그램이 있는지도 알고 싶습니다.
